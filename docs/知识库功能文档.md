## 功能概述

### 什么是知识库？

知识库功能允许你将**本地文档**（Markdown、Word、PDF、TXT 等）上传到系统中，系统会自动：

1. **解析文档内容**（支持 .md, .txt, .docx, .pdf, .html, .epub）
2. **智能分块**（滑动窗口策略，保留上下文）
3. **向量化索引**（使用 Embedding 模型将文本转为向量）
4. **语义搜索**（根据语义相似度检索相关内容）
5. **RAG 对话**（检索增强生成，让 AI 基于你的文档回答问题）

### 核心应用场景

- 📚 **技术文档问答**：上传 API 文档、设计文档，直接问问题
- 📝 **会议记录检索**：上传会议记录，快速查找关键决策
- 🔬 **研究论文管理**：上传论文 PDF，语义搜索相关段落
- 📖 **知识沉淀**：团队积累的知识库，随时查询

## 技术栈

### 后端

| 组件 | 技术选型 | 版本 | 用途 |
|------|---------|------|------|
| **向量数据库** | ChromaDB | 1.5.9 | 存储文本块的向量和元数据 |
| **Embedding 模型** | nomic-embed-text | - | 文本向量化（768 维） |
| **LLM** | Qwen2.5-3B (Ollama) | - | RAG 对话生成回答 |
| **文档解析** | PyMuPDF, python-docx, BeautifulSoup | - | 多格式文档加载 |
| **Web 框架** | Flask / FastAPI | - | API 服务 |

### 前端

| 组件 | 技术选型 | 用途 |
|------|---------|------|
| **UI 框架** | 原生 HTML + CSS + JavaScript | 轻量级，无框架依赖 |
| **样式** | CSS3 (CSS 变量主题) | 深色/浅色主题切换 |
| **HTTP 请求** | Fetch API | 与后端通信 |

### 核心算法

- **文本分块**：滑动窗口（Sliding Window）+ 递归字符分割
- **向量检索**：余弦相似度（Cosine Similarity）
- **RAG 提示词**：检索结果 + 用户问题 → LLM 生成回答

## 功能详解

### 1. 知识库初始化

**目的**：加载 Embedding 模型、初始化 ChromaDB、准备 RAG 链

**API 端点**：`POST /api/kb/init`

**参数**：
```json
{
  "chunk_size": 500,        // 分块大小（字符数）
  "overlap": 50,            // 重叠字符数（保留上下文）
  "embed_provider": "ollama",  // Embedding 提供者
  "embed_model": "nomic-embed-text"  // Embedding 模型名称
}
```

**初始化流程**：
1. 检查 ChromaDB 持久化目录是否存在
2. 加载 Embedding 模型（Ollama 本地服务）
3. 初始化向量数据库客户端
4. 创建 RAG 链（复用现有 summarizer）
5. 返回统计信息（已有文档数、块数）

**注意**：初始化状态存储在内存中，ai_worker 重启后需重新调用此 API。

---

### 2. 文档上传与索引

**目的**：上传文档并自动索引到向量数据库

**API 端点**：`POST /api/kb/upload`

**请求格式**：`multipart/form-data`
- `file`: 文档文件（支持 .md, .txt, .docx, .pdf, .html, .epub）
- `category`: 文档分类（可选，默认 "默认"）

**处理流程**：
```
上传文件
  ↓
保存到临时文件
  ↓
DocumentLoader 解析文档
  ↓
TextChunker 分块（滑动窗口）
  ↓
Embedder 向量化每个块
  ↓
VectorStore 存储到 ChromaDB
  ↓
删除临时文件
  ↓
返回文档 ID 和块数
```

**支持的文档格式**：
- ✅ `.md` - Markdown（保留格式）
- ✅ `.txt` - 纯文本
- ✅ `.docx` - Word 文档（提取段落）
- ✅ `.pdf` - PDF 文档（按页提取）
- ✅ `.html` - HTML 网页（去除标签）
- ✅ `.epub` - 电子书

---

### 3. 语义搜索

**目的**：根据语义相似度检索相关文本块

**API 端点**：`GET /api/kb/search?q=<查询>&top_k=<数量>`

**参数**：
- `q` (必需): 搜索查询（自然语言）
- `top_k` (可选): 返回结果数量，默认 5
- `category` (可选): 按分类过滤

**搜索流程**：
```
用户输入查询
  ↓
Embedder 向量化查询
  ↓
VectorStore 余弦相似度检索
  ↓
返回 top-k 个最相关的文本块
  ↓
包含：文件名、分类、文本内容、相似度分数
```

**返回示例**：
```json
{
  "query": "LangChain 和 Open WebUI 的区别",
  "total": 3,
  "results": [
    {
      "id": "xxx_c14",
      "filename": "open_webui_vs_langchain.md",
      "category": "开发对比",
      "text": "LangChain 是一个开发者工具库...",
      "chunk_index": 14,
      "score": 0.8234,
      "distance": 0.1766
    }
  ]
}
```

---

### 4. RAG 对话

**目的**：结合知识库内容回答用户问题

**API 端点**：`POST /api/kb/chat`

**请求格式**：
```json
{
  "question": "LangChain 和 Open WebUI 有什么区别？",
  "top_k": 5,
  "filter_category": null,
  "model_id": null
}
```

**RAG 流程**：
```
用户提问
  ↓
① 检索：语义搜索 top-k 相关文本块
  ↓
② 构建提示词：
  """
  你是知识库助手。根据以下参考文档回答问题。
  
  参考文档：
  1. [文件名] 文本内容...
  2. [文件名] 文本内容...
  
  问题：LangChain 和 Open WebUI 有什么区别？
  """
  ↓
③ 生成：调用 LLM (summarizer.chat())
  ↓
④ 返回：回答 + 来源文档列表
```

**返回示例**：
```json
{
  "answer": "LangChain 和 Open WebUI 的主要区别如下：\n1. 定位不同...\n2. 使用方式...",
  "sources": [
    {
      "filename": "open_webui_vs_langchain.md",
      "category": "开发对比",
      "text": "LangChain 是一个开发者工具库...",
      "score": 0.8234
    }
  ]
}
```

---

### 5. 文档管理

#### 列出所有文档

**API 端点**：`GET /api/kb/docs`

**返回**：已索引的所有文档列表（去重，按 doc_id 分组）

```json
{
  "docs": [
    {
      "doc_id": "3fc2b3fc0fc8",
      "filename": "open_webui_vs_langchain.md",
      "chunk_count": 25,
      "category": "开发对比",
      "indexed_at": "2026-06-22T12:00:00"
    }
  ],
  "count": 2
}
```

#### 删除文档

**API 端点**：`DELETE /api/kb/doc/{doc_id}`

**效果**：从 ChromaDB 中删除该文档的所有文本块

#### 清空知识库

**API 端点**：`DELETE /api/kb/clear`

**效果**：删除 ChromaDB 中所有数据（不可恢复）

## 使用指南

1. 在 **🔍 语义搜索** 输入框输入关键词

1. 在 **💬 RAG 对话** 输入框输入问题
2. 点击 **➡️ 发送** 或按 **Enter**

**示例对话**：
```
🧑 你：LangChain 和 Open WebUI 有什么区别？

🤖 AI：LangChain 和 Open WebUI 的主要区别如下：
1. **定位不同**：LangChain 是开发者工具库，Open WebUI 是终端用户产品

📎 参考来源：open_webui_vs_langchain.md、agent_tool_comparison.md
```

#### 5. 管理文档

- **查看列表**：页面加载时自动显示所有已索引文档
- **删除文档**：点击文档右侧的 **✕** 按钮
- **清空知识库**：点击顶部 **🗑️ 清空** 按钮（需确认）

```bash
curl -X DELETE "http://127.0.0.1:8000/api/kb/clear"
```

#### Python 示例

```python
import requests

BASE_URL = "http://127.0.0.1:8000/api/kb"

# 1. 初始化
response = requests.post(f"{BASE_URL}/init", json={
    "chunk_size": 500,
    "overlap": 50
})
print(response.json())

# 2. 上传文档
with open("document.md", "rb") as f:
    response = requests.post(f"{BASE_URL}/upload", files={"file": f}, data={"category": "技术文档"})
print(response.json())

# 3. 语义搜索
response = requests.get(f"{BASE_URL}/search", params={"q": "LangChain RAG", "top_k": 5})
results = response.json()["results"]
for r in results:
    print(f"📄 {r['filename']} (相似度: {r['score']:.2f})")
    print(f"   {r['text'][:100]}...")

# 4. RAG 对话
response = requests.post(f"{BASE_URL}/chat", json={
    "question": "LangChain 支持哪些向量数据库？"
})
data = response.json()
print(f"🤖 AI：{data['answer']}")
print(f"📎 来源：{', '.join(s['filename'] for s in data['sources'])}")

# 5. 列出文档
response = requests.get(f"{BASE_URL}/docs")
docs = response.json()["docs"]
for d in docs:
    print(f"📄 {d['filename']} ({d['chunk_count']} 块)")

# 6. 删除文档
doc_id = docs[0]["doc_id"]
requests.delete(f"{BASE_URL}/doc/{doc_id}")
```

## API 文档

### 端点汇总

| 方法 | 端点 | 功能 | 请求体 | 响应 |
|------|------|------|--------|------|
| POST | `/api/kb/init` | 初始化知识库 | JSON | `{status, stats}` |
| POST | `/api/kb/upload` | 上传文档 | FormData | `{status, doc_id, chunk_count}` |
| POST | `/api/kb/index` | 批量索引目录 | JSON | `{status, indexed, failed}` |
| GET | `/api/kb/docs` | 列出文档 | - | `{docs: [], count}` |
| GET | `/api/kb/stats` | 统计信息 | - | `{total_chunks, total_docs}` |
| GET | `/api/kb/search` | 语义搜索 | Query | `{query, total, results}` |
| POST | `/api/kb/chat` | RAG 对话 | JSON | `{answer, sources}` |
| DELETE | `/api/kb/doc/{id}` | 删除文档 | - | `{status, message}` |
| DELETE | `/api/kb/clear` | 清空知识库 | - | `{status, message}` |
| GET | `/api/kb/status` | 获取状态 | - | `{initialized, stats}` |

### 详细参数说明

#### POST /api/kb/init

**请求参数**：
- `chunk_size` (int, 默认 500): 分块大小（字符数），范围 100-2000
- `overlap` (int, 默认 50): 重叠字符数，范围 0-500
- `embed_provider` (str, 默认 "ollama"): Embedding 提供者（ollama/huggingface）
- `embed_model` (str, 默认 "nomic-embed-text"): Embedding 模型名称

**响应**：
```json
{
  "status": "ok",
  "message": "知识库初始化成功",
  "stats": {
    "total_chunks": 41,
    "total_docs": 2
  }
}
```

**错误响应**：
```json
{
  "detail": "知识库初始化失败: Ollama 服务未启动"
}
```

---

#### POST /api/kb/upload

**请求格式**：`multipart/form-data`

**表单字段**：
- `file` (File, 必需): 文档文件
- `category` (String, 可选): 文档分类，默认 "默认"

**响应**：
```json
{
  "status": "ok",
  "message": "文档 'open_webui_vs_langchain.md' 索引成功",
```

**错误响应**：
- `400`: 不支持的文件格式 / 文件名为空
- `500`: 索引失败（解析错误、Embedding 失败等）

---

#### GET /api/kb/search

**查询参数**：
- `q` (String, 必需): 搜索查询（最小长度 1）
- `top_k` (int, 默认 5): 返回结果数量，范围 1-20
- `category` (String, 可选): 按分类过滤

**响应**：
```json
{
  "query": "LangChain RAG",
  "total": 3,
  "results": [
    {
      "id": "xxx_c14",
      "filename": "open_webui_vs_langchain.md",
      "category": "开发对比",
      "text": "LangChain 提供了 RAG 支持...",
      "chunk_index": 14,
      "score": 0.8234,
      "distance": 0.1766
    }
  ]
}
```

**字段说明**：
- `score`: 相似度分数（0-1，越高越相关）
- `distance`: 距离分数（越低越相关）
- `chunk_index`: 文本块在原文中的序号

---

#### POST /api/kb/chat

**请求体**：
```json
{
  "question": "LangChain 支持哪些向量数据库？",
  "top_k": 5,
  "filter_category": null,
  "model_id": null
}
```

**字段说明**：
- `question` (String, 必需): 用户问题
- `top_k` (int, 默认 5): 检索的上下文数量
- `filter_category` (String, 可选): 按分类过滤
- `model_id` (String, 可选): 指定 LLM 模型（复用 ai_worker 的模型切换）

**响应**：
```json
{
  "answer": "LangChain 支持多种向量数据库，包括：\n1. ChromaDB\n2. FAISS\n3. Pinecone\n...",
```

## 技术实现细节

### 1. 文本分块策略

**问题**：长文档直接向量化会丢失上下文，且超过 Embedding 模型的最大长度限制。

**解决方案**：滑动窗口分块

```python
def chunk_text(text, chunk_size=500, overlap=50):
    chunks = []
    start = 0
    while start < len(text):
        end = start + chunk_size
        chunk = text[start:end]
        chunks.append(chunk)
        start += (chunk_size - overlap)  # 滑动窗口
    return chunks
```

**示例**：
```
原文： "LangChain 是一个框架。它支持 RAG。RAG 很重要。"
分块（chunk_size=20, overlap=5）：
  - 块1: "LangChain 是一个框架。它支持"
  - 块2: "它支持 RAG。RAG 很重"
  - 块3: "RAG 很重要。"
```

**优点**：
- 保留上下文（重叠部分）
- 适配 Embedding 模型输入长度
- 检索时返回相关片段，而非整篇文档

### 2. 向量化与检索

**Embedding 过程**：
```
文本块: "LangChain 是一个框架..."
  ↓
Ollama API: POST /api/embeddings
  {
    "model": "nomic-embed-text",
    "prompt": "LangChain 是一个框架..."
  }
  ↓
返回: [0.123, -0.456, ..., 0.789]  (768 维向量)
  ↓
存储到 ChromaDB: collection.add(
    embeddings=[[...], [...], ...],
    documents=["文本1", "文本2", ...],
    metadatas=[{"filename": "...", ...}, ...],
    ids=["xxx_c0", "xxx_c1", ...]
)
```

**检索过程**：
```
用户查询: "LangChain 支持哪些数据库？"
  ↓
向量化: Embedder.embed_query(query) → [0.234, -0.567, ..., 0.891]
  ↓
ChromaDB 检索: collection.query(
    query_embeddings=[query_vector],
    n_results=top_k
)
  ↓
返回: [
    {"id": "xxx_c14", "document": "...", "distance": 0.1766, "metadata": {...}},
    ...
]
```

**相似度计算**：余弦相似度（Cosine Similarity）
```
similarity = cos(θ) = (A · B) / (||A|| * ||B||)
```
- `score = 1 - distance`（转换为 0-1 范围）
- `score` 越高，语义越相关

### 4. 前端标签页切换实现

**问题**：之前的标签页按钮（工作站、AI 助理、文档）没有绑定点击事件，点击无反应。

**解决方案**：事件委托（Event Delegation）

```javascript
const tabBar = document.querySelector('.tab-bar');
const tabPages = {
    workstation: document.getElementById('page-workstation'),
    assistant: document.getElementById('page-assistant'),
    docs: document.getElementById('page-docs'),
    knowledge: document.getElementById('page-knowledge'),
};

// 事件委托：在父元素上监听点击，判断点击目标
tabBar.addEventListener('click', (e) => {
    const btn = e.target.closest('.tab-btn');  // 找到最近的 .tab-btn
    if (!btn) return;  // 点击的不是按钮，忽略
    
    const tab = btn.dataset.tab;  // 获取 data-tab 属性
    
    // 1. 切换按钮 active 状态
    tabBar.querySelectorAll('.tab-btn').forEach(b => b.classList.remove('active'));
    btn.classList.add('active');
    
    // 2. 切换页面显示
    Object.keys(tabPages).forEach(key => {
        if (tabPages[key]) {
            tabPages[key].classList.toggle('active', key === tab);
        }
    });
});
```

**为什么用事件委托？**
- ✅ 动态添加的标签页也能生效（如未来新增"设置"标签）
- ✅ 减少事件监听器数量（性能优化）
- ✅ 代码简洁

## 常见问题

### Q1: 初始化时报错 "Ollama 服务未启动"

**错误信息**：
```json
{"detail": "知识库初始化失败: Ollama 服务未启动"}
```

**解决方法**：
1. 检查 Ollama 是否运行：
   ```bash
   ollama list
   ```
2. 如果未运行，启动 Ollama：
   ```bash
   ollama serve
   ```
3. 确认 Embedding 模型已拉取：
   ```bash
   ollama pull nomic-embed-text
   ```

---

### Q2: 上传文档时报错 "不支持的格式"

**错误信息**：
```json
{"detail": "不支持的格式: .xlsx，支持的格式: .md, .txt, .docx, .pdf, ..."}
```

**解决方法**：
- 目前支持的格式：`.md`, `.txt`, `.docx`, `.pdf`, `.html`, `.epub`
- 如果需要支持其他格式，可以：
  1. 手动转换为支持的格式
  2. 或者修改 `knowledge_store.py` 中的 `SUPPORTED_EXTENSIONS`

---

### Q3: 语义搜索结果不相关

**可能原因**：
1. **文档内容确实不相关**：尝试调整查询关键词
2. **分块不合理**：调整 `chunk_size` 和 `overlap`
3. **Embedding 模型不适合**：尝试更换为其他 Embedding 模型

**解决方法**：
- 调整 `chunk_size`（默认 500）：
  - 增大：保留更多上下文，但可能包含无关内容
  - 减小：更精准，但可能丢失上下文
- 调整 `top_k`（默认 5）：
  - 增大：检索更多候选，但可能包含不相关结果
  - 减小：更精准，但可能漏掉相关信息

---

### Q4: RAG 对话回答不准确

**可能原因**：
1. **检索到的上下文不相关**：见 Q3
2. **LLM 模型能力不足**：尝试切换更强的模型
3. **提示词不够明确**：可以自定义 RAG 提示词模板

**解决方法**：
- 切换 LLM 模型（在 `config.yaml` 中配置）
- 调整 `top_k`（检索更多上下文）
- 修改 `knowledge_store.py` 中的 `RAGChain.DEFAULT_PROMPT_TEMPLATE`

---

### Q5: ai_worker 重启后知识库数据丢失？

**现象**：
- 重启 ai_worker 后，调用 `GET /api/kb/stats` 返回 `total_chunks: 0`
- 但 ChromaDB 持久化目录（`knowledge_base/chroma_db/`）中还有数据

**原因**：
- 知识库的初始化状态（`_initialized`）存储在内存中
- ai_worker 重启后，需要重新调用 `POST /api/kb/init` 恢复状态

**解决方法**：
1. 手动调用初始化 API：
   ```bash
   curl -X POST "http://127.0.0.1:8000/api/kb/init"
   ```
2. 或者修改 `knowledge_api.py`，在启动时自动检测并初始化：
   ```python
   @router.on_event("startup")
   async def auto_init():
       if os.path.exists(KB_ROOT / "chroma_db"):
           init_knowledge_base(...)
   ```
