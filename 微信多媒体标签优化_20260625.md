## 问题背景

群聊分析页面中图片/语音/文件/视频消息使用大卡片展示（120×120px 占位符、语音波形动画等），但由于微信 3.x 的 `.dat` 文件无法解密，卡片永远只显示占位符，反而浪费大量视觉空间。

**根因**: 微信 3.x 使用未知复杂加密算法，无公开解密方法（`07 08 56 32 ...` 头部，非简单 XOR）。

---

## 解决方案

### 设计原则
> 不展示无法获取的内容，用紧凑形式表达，给用户快速信息 + 便捷操作入口。

### 新 UI 方案

**行内标签**（在消息气泡内）：
```
🖼️ 图片 #1234    🎤 语音 15"    📎 会议记录.pdf    🎬 视频 2'34"
```
- 紧凑内联，不占额外高度
- 带颜色区分（紫色/蓝色/绿色/橙色）
- 悬停放大，点击弹出详情

**详情浮层**（点击后）：
```
┌─────────────────────────────┐
│  ✕                           │
│  🖼️ 图片消息                  │
│                              │
│  标识ID      #1234            │
│  原始数据    [图片] (local_i..│
│                              │
│  [📋 复制信息]  [关闭]        │
│                              │
│  💡 图片以加密格式存储在本地，  │
│     需在微信客户端中打开查看。 │
└─────────────────────────────┘
```

## 技术实现

### 新增函数
| 函数 | 用途 |
|------|------|
| `renderMediaTag(type, meta)` | 生成行内标签 HTML |
| `showMediaPopover(event, type, meta)` | 显示详情浮层 |
| `closeMediaPopover()` | 关闭浮层 |
| `copyMediaInfo(type, raw)` | 复制媒体原始信息到剪贴板 |
| `showToast(msg, type)` | Toast 提示 |

### 删除内容
- `renderImageMessage` / `showImagePlaceholder`
- `renderVoiceMessage` / `toggleVoicePlayback`
- `renderFileMessage` / `showFileDownloadTip`
- `renderVideoMessage` / `showVideoPlaceholder`
- 所有旧大卡片 CSS（`.wechat-image-item` / `.wechat-voice-item` / `.wechat-file-item` / `.wechat-video-item` 等）
- 旧图片预览模态框（`wechat-image-modal`）

### CSS 类
| 类名 | 用途 |
|------|------|
| `.wechat-media-tag` | 标签容器 |
| `.media-tag-image/voice/file/video` | 类型样式 |
| `.wechat-media-popover` | 浮层背景 |
| `.popover-title/meta/actions/hint` | 浮层内部结构 |

## 微信3.x解密进展

- **已确认**: wechat-cli 源码只有数据库解密（SQLCipher），无图片解密
- **尝试过的方法**: XOR 0x00-0xFF、跳过前N字节、XOR多字节key → 全部失败
- **结论**: 短期内无公开解密方案，建议保持现状
