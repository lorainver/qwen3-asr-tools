二、编程升学的优势与劣势

2.1 核心优势

最大优势：编程达到一定水平（四级/五级左右）的学生，可以获得远超纯文化课路径的学校考试机会。
  • 考试机会充足：编程五级以上的学生，"基本上每一个娃儿都是把七龙珠挨着考了一遍"，只有少数学校未参加。学生"不缺考试机会"。
  • 突破文化课考试机会稀缺的瓶颈：当前政策环境下，纯文化课考核的考试机会"非常少"。大部分学习奥数的学生"没得机会进入这些学校的选拔范围"，而编程五级水平即可获得这些学校的考核资格。
  • 考核内容与最终轮次等效：编程考试获得的考核机会，与进入最终轮次的文化课考核"没啥子区别的"。
  • 录取通道更直接：以一中为例，通过编程路径只需关注"编程加数学两科"，即可在一轮考核中锁定本部班型，流程远少于纯文化课路径。

2.2 核心劣势

最大劣势：学习习惯不达标的学生，即使编程能力很强，也可能在"跟训"环节被淘汰。
  • 跟训淘汰机制严格：除一中以外，几乎所有目标校（育才、南开、八系、西附、一外等）都设有跟训环节。跟训期间会综合考察学生的学习习惯和家长的配合度。
  • 家长因素直接影响结果：存在"孩子打五分，家长被打负分"而被剔除的案例。学校在跟训环节"也是在面试家长"，通情达理、配合度高的家长更受认可。
  • 能力与习惯缺一不可："你但凡学习习惯偏一点，是没得办法拿到签约机会的。"

2.3 编程升学的双面性总结

维度 优势面 劣势面
考试机会 机会充足，七龙珠几乎全覆盖 考核标准高，需要真正达到相应水平
录取通道 流程更短、更直接 部分学校仍需加考文化课
签约保障 考核通过即可锁定班型 跟训环节存在淘汰风险
家庭投入 竞赛路径可避开纯文化课的激烈竞争 学习习惯培养极其"费家长"

三、各目标校的选拔流程与考察侧重

3.1 总体说明

各校在编程升学考核中的考察方向、流程安排、文化课权重均存在差异。家长需根据目标校的不同，有针对性地调整训练重心。

重要提示："你不能说考了几级几级就怎样怎样，它不一样的，针对不同学校，你就得去准备。"

3.2 各校选拔流程与考察偏好

南开（三系） 数学思维测评 → 编程考核/跟训 → 签约 数学底层逻辑思维（非小奥刷题方向），编程应用能力要求高 有数学思维测评（现学现考形式）

3.3 各校考察内容分类

思维类考察（南开三系、八系）

  • 重点考察数学底层逻辑思维，与小奥刷题关联不大。
  • 包含内容：递推、二分、贪心（限享策略）等思维体的综合训练。
  • 南开特别强调数学逻辑能力，"跟刷小奥没得关系"，建议提前学习初联内容。

算法类考察（一中、西附、一外）

  • 侧重算法模板的掌握与应用。
  • 建议算法水平达到六级，将整个普及组学完较为稳当。

纯思维类考察（育才）

    考察纯思维题，不设文化课考试，主要根据编程跟训表现签约。 跟训签约周期相对较长。

四、跟训机制详解

4.1 跟训的本质

"跟训"是多数七龙珠学校在编程考核通过后设置的综合考察环节，本质上是学校对学生和家庭的进一步筛选。

4.2 跟训考察的两个维度

  1. 学生维度——学习习惯：
    • 即使编程能力很强，学习习惯不好也会被刷掉。
    • 存在"资质较好但因学习习惯卡在跟训环节被踢掉"的案例。
    • 学校非常看重学生的自主学习能力、作业完成质量、课堂表现等。
  2. 家长维度——配合度与沟通能力：
    • 学校会在跟训环节与家长沟通，实质上"也是在面试家长"。
    • "听话的家长"更受欢迎；提出过多不合理要求的家长可能影响孩子录取。
    • 极端案例："孩子打五分，家长被打负分，然后就被剔除掉了。"

4.3 各校跟训情况对比

学校 是否有跟训 备注
一中 有 也有跟训环节，并非例外
南开三系 有 数学思维测评 + 编程跟训
八系 有 跟训 + 文化课 + 现学现考，考察最全面
育才 有 无文化课考核，但跟训时间相对最长
西附 有（2025年新增） 此前无跟训，今年新增

趋势观察：越来越多的学校在编程升学路径中增设跟训环节，这意味着即便编程能力达标，学生的综合学习习惯和家庭配合度已成为不可忽视的硬性门槛。

五、训练策略与准备方式

5.1 针对不同目标校的训练重点

目标校 训练重点 具体建议
一中 算法模板 可压缩校内时间，集中精力在编程加数学两科；进入竞赛体系后，竞赛和文化课均需校外预习
南开 数学底层思维 + 高水平编程 编程达六级以上；数学建议提前学习初联（替代小奥刷题）；核心看逻辑思维能力
八系 思维体综合训练 需专项训练递推、二分、贪心（限享策略与优化能力）；考察最全面
育才 纯思维题 可甩掉校内负担，全力投入编程训练；进入竞赛体系后需校外预习
西附 / 一外 算法水平 算法水平建议达到六级，学完整个普及组

5.2 思维训练补充方案

若学生所在机构未提供专门的思维训练课程，分享人建议通过以下方式进行补充：

AtCoder Beginner Contest（ABC）训练

  • 频率：每周六晚上进行。
  • 适用水平：考级达到四级即可开始ABC训练。
  • 补题要求："不补题、不写总结，是永远不会进步的。"只能做三题的学生，需要通过题解将剩余题目补完，最好有老师讲解思路。
  • 推优门槛：ABC Rating达到1000以上，可获得向金刚校推优的机会。

ABC翻译工具

  • 分享人推荐使用特定翻译工具（由合作老师开发），具备后台人工审核机制，翻译准确度高于AI自动翻译。
  • 该翻译网站同时提供当周ABC题解，便于学生补题学习。

思维板块专项课程

  • 分享人所在机构设有专门的思维板块专项训练课程。
  • 课程性质为"方法课"而非知识点课，教授内容包括：如何写题解与总结、如何手推样例、如何用数学方法解决算法性问题等。

5.3 编程与数学的时间分配建议

核心建议：数学与编程"五五开"。
  • 理想状态：数学和编程水平都强，兼顾发展。
  • 时间不足时：压缩校内时间，优先保障编程和数学训练。
  • 走单科路线：若完全放弃文化课路径、单攻编程方向，编程水平需达到七八级。
  • 数学对编程的重要性："学数学对于编程很重要"，但不建议盲目刷小奥题。数学训练应有"方式方法"，注重底层思维而非题量。
  • 关于现学现考：编程水平高的学生在现学现考中"不太怕"，因为他们的学习内容本身就很超前，思维调用率高。

六、南开三系竞赛班体系详解

6.1 三系竞赛梯队构成

校区 班型 人数规模 状态
南开本部 秋班（竞赛班） 约30人/班 成熟运行
南渝 竞赛行政班 约40–50人/班 成熟运行（含竞赛生 + 综合优秀生）
融侨 信竞行政班 约40–50人/班 成熟运行
两江南开 竞赛班型 待定 体系正在建设中

6.2 秋班选拔机制

  • 选拔时间：六年级的七、八月份（即将升入初一阶段）。
  • 考核内容：数学（初联难度）+ 编程。
  • 人员来源：所有分校竞赛生均需通过此最终考核，才能进入本部秋班。
  • 留存机制：未进入秋班的学生，回到学籍所在地的分校竞赛班。

6.3 竞赛班总体规模

南开三系整个竞赛体系的学生总量约为120–150人。值得注意的是，"整个重庆能达到人家那个标准的学生，五十人都没得"，因此学校常出现招不满的情况，很多文化课上岸的学生后续转入编程竞赛方向。

6.4 内招与外招

  • 外招：通过编程考核直接招收的学生（名额有限）。
  • 内招：文化课上岸后，参加学校编程零基础培训，再进入竞赛班。
  • 内招存在的原因："外招招不起那么多人"，全市达到标准的学生总量不足，无法满足学校招生体量。

七、区县学生升学机会

7.1 区县学生的优势

重庆主城各校（尤其是金刚校）对区县学生"非常喜欢"。区县学生由于未经历主城小奥的过度开发，往往处于"比较原始的状态"，许多学校偏好这种"原始的牛娃"。

7.2 区县学生的限制

  • 跨区难度增大："现在重庆主城要跨区，都需要做实质性的贡献才行了。"学校想要，但教委政策限制。
  • 编程水平门槛：区县学生需达到一定的竞赛层次才能获得跨区机会。
  • 资源有限：区县编程培训资源相对匮乏，学生编程水平可能受限于学习环境。

7.3 区县学生的机会判断

分享人提供了一个评估路径：

  1. 思维卷测试：通过内测思维卷评估学生的底层数学思维能力。
  2. 推优门槛：思维卷得分达到70分以上，可获得推优至金刚校的机会。
  3. 编程考级辅助：若编程水平暂未达标，但思维能力突出，仍有机会。
  4. 备选方案：若无法走金刚校路径，可冲刺"4SP"（信息学竞赛）的普及组一等成绩，获得翻墙考的机会。

7.4 区县学生的翻墙机会

若区县学生想通过三系（南开体系）翻墙，只有一次机会：在六年级升初一阶段，4SP（信息学竞赛）达到普及组一等以上成绩，学校可能给予一次"反抢考"机会。但"必须达到这个层次才行，没有这个层次的话是没有学校给机会的"。

八、编程学习起步时间与进度规划

8.1 建议起步时间

时间节点 建议
最佳启蒙期 一年级下学期（时间最充裕）
最晚进入期 不晚于四年级（最迟不超过五年级）
小升初集中冲刺期 五年级下册至六年级上册（机会最集中的阶段）

8.2 各年级进度参考

  • 一年级：启蒙阶段，培养兴趣与基础认知。
  • 二年级下学期：优秀学生可完成算法课储备，进入C++学习。
  • 四年级：考级达到四级后，可开始ABC（AtCoder）训练。
  • 五年级寒假起：升学机会开始增多，编程五级可获得大量考试资格。
  • 五年级下册–六年级上册：上岸的"比较集中的时间"。
  • 六年级：冲刺阶段，南开秋班考核在六年级七八月份。

8.3 进度追赶建议

对于起步较晚的学生（如五年级才过二级）：

  • 利用暑假"拉一拉进度"。
  • 六年级参加考试。
  • 不必过度焦虑，编程考试主要看数学现学现考成绩，"现学现考，你这个小娃子再飞起来也没用"，编程水平高的学生在此环节有天然优势。

九、家长配合的关键要素

9.1 家长配合度的核心价值

分享人以一个成功案例说明：家长"非常听话"是关键因素。该学生家长在周老师安排课程时，从不质疑决策（如被建议学习斯库奇算法课时，未提出"这是不是智商税"的疑问），严格执行规划，最终该学生二年级下学期通过算法课储备进入C++学习，一路升入金刚校全托班。

9.2 家长常见问题

  • 过度质疑规划："家长天天质疑你的决策，那就没法玩了。"家长信息不全面，而规划老师信息更全面，若不信任专业判断，将错失机会。
  • 不重视学习习惯：学习习惯的培养"非常费家长"，"很多妈经不起费"。若家长不重视，孩子的习惯提升将停滞。
  • 提出不合理要求："不要提些奇奇怪怪的要求"，通情达理是基本要求。

9.3 学习习惯的培养

分享人明确指出：学习习惯的调整不是培训机构能单独解决的，它与家庭环境和家长要求高度相关。孩子的学习习惯可能要到初三之后才能定型。

  • 编程作业需要动用大脑80%–100%的调用率，而校内作业仅需约50%。
  • 若不持续督促，孩子的思维调用率会迅速回落至50%的"舒适区"。
  • "你不激他，他都马上掉回百分之五十。"

十、编程学习的延伸价值

10.1 升学之外的长远价值

分享人强调，编程学习的作用远不止于小升初升学：

  • AI与大模型学习：分享人正在学习AI智能体课程，发现"没有编程基础学那个大模型的课，学不懂"，有编程基础的学生在理解大模型相关课程时明显更具优势。
  • 理科能力培养：编程特长生的理科能力普遍较强，这也是各校愿意招收编程方向学生的原因之一——学校选拔编程特长生"不是为了让你去搞竞赛，也是为了让你去搞综合"。

十一、常见问题解答（精选）

Q1：编程几级可以开始考学校？

四级/五级左右即可获得大量考试机会。五级以上基本可以"把七龙珠挨着考一遍"。

Q2：编程考级等级是否等同于思维赛水平？

不一定。考级等级（六级、七级、八级）反映的是知识体系掌握程度，而思维赛（如ABC）的Rating反映的是解题能力，两者衡量维度不同。

Q3：如何判断孩子是否适合走编程升学？

  • 需具备一定的数学底层思维能力（可通过内测思维卷评估，70分以上为及格线）。
  • 学习习惯必须达标（跟训环节的硬性门槛）。
  • 家长需具备配合度与信任度。

Q4：文化课和编程如何平衡？

建议"五五开"。若时间不够，压缩校内时间，优先保障编程和数学。对于有志于竞赛班的学生，分享人建议"小学下午就不要上了，把时间拿出来冲班"。

Q5：区县学生如何获得机会？

三条路径：

  1. 通过内测思维卷（70分以上）获得推优资格。
  2. 通过4SP普及组一等成绩获得翻墙考机会。
  3. 持续提升编程水平，等待学校主动邀约。

Q6：沙巴（沙坪坝八中）如何进入？

需先进入红巴（宏巴）的竞赛班，在其中的优秀学生可提前进入三系竞赛体系。沙巴的竞赛班主要面向初中生，选拔后可直接衔接八中竞赛梯队。
