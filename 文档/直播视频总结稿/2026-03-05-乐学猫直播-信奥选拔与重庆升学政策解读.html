CHAPTER 01各校选拔动态与寒假变化

1.1 背景概述

2024年10—11月，某校组织了一次大规模编程选拔活动，考试难度极高。此后进入政策敏感期，部分学校取消了所有培训活动，导致学生大量流失。至寒假期间（12月），原校开班时已面临严重生源缺口。

关键事件 敏感期期间，该校全面停训两个月，学生被其他学校"挖走"殆尽。据反映，仅剩个别学生因适应问题回流。这一连锁反应直接催生了后续各校的大规模扩招。

1.2 寒假各校动向

金刚校 A（跟训+文化课模式）

  • 寒假安排15天跟训（年前10天 + 年后5天），跟训结束后进行编程结业考试与文化课考试
  • 最终编程定签约，数学定进入哪个班型
  • 复试通知以电话形式发送，出现大量家长误认为骚扰电话直接挂断的情况

金刚校 B（新开编程招生）

  • 2025年为第一届正式招收编程学生，此前信奥招生人数极少
  • 寒假考试分两批：五年级一批、六年级一批，分开考核
  • 录取学生进入本部竞赛顶班，实行五年级/六年级脱产制度——学籍留在原年级，实际在初一、初二就读，提前获得多年备战时间
  • 编程学生招生不看年级，四、五、六年级拉通考试，只要水平达标即统一推荐
  • 首届招生名额：一届80人，本部就读，且各分校上岸学生均可签约

金刚校 C（育才模式 / 跟学制）

  • 采用新筛选模式：先在机构内进行笔试内测，达标后由学校自行筛选
  • 内测通过的学生进入每周考核机制——考思维题、评奖、补贴
  • 该校模式对低龄段学生非常友好：不以当前编程水平为唯一标准，更看重长期表现、学习习惯、习题表现与代码反馈
  • 本次内测中，某机构推荐的约20名学生被选中（占该班近半数），其中包含二级、三级学生
寒假整体趋势 各校因国家队入选事件与扩招需求，在寒假期间疯狂招生，导致整个寒假成为近年来最忙碌的招生季。频繁出现名单整理、活动通知、学校对接等高强度工作。

CHAPTER 02信息学竞赛划线逻辑解析

核心认知纠偏 重庆信息学分数线高、获奖人数少，根本原因并非"太卷"，恰恰是参与人数太少。

2.1 划线机制

信息学竞赛的分数线划定逻辑与常规文化课不同，核心特征为按参赛基数的比例划线：

  • 省队名额：与参赛基数挂钩，参赛人数越多，分到的省队名额越多
  • 省赛一等奖：同样按比例划定，参赛基数越大，一等奖获奖人数越多
  • 重庆每年省赛参赛仅数百人，但省队已有十几个名额，对其他省市而言已属资源倾斜
逻辑反转 参与人数越多 → 分数线反而越低 → 省赛一等奖名额越多。
参与人数越少 → 按比例划线后，参赛者全是高手 → 分数线被推至极高 → 获奖人数极少。因此，更多人参与信息学竞赛，对整体生态是有利的。

2.2 分数线参考

以提高组为例，每年省赛一等奖仅四十余人。某校教练指出，学校方面已经在降低要求，但学生基数不足仍是制约获奖人数的核心因素。

CHAPTER 03各校选拔模式对比分析

3.1 三种典型选拔模式

维度 模式一：跟训签约制 模式二：编程+文化课分层 模式三：内测+长期跟踪
考核方式 长期跟训 → 排名/积分 → 签约 编程考试 + 文化课考试，分层筛选 机构内测 → 每周考核思维题 → 学校复筛
周期 较长，需在跟训中积累资历与排名 寒假集中15天跟训 + 结业考试 滚动筛选，周期相对灵活
侧重点 编程综合排名 编程定签约 + 数学定班型 长期表现 > 单次成绩
低龄段友好度 较低（需熬资历） 中等（五年级即有机会） 高（可辐射三、四年级）
签约速度 慢 较快 中等（培优班可加速）

3.2 各校考试内容偏好差异

注意 每所学校对学生的偏好存在差异，考试内容与选拔标准并不统一。部分学校重视编程能力，部分学校更看重综合素养。小升初过程中存在一定的主观判断与运气成分。

金刚校 A 的筛选特点

  • 编程通过后，复试文化课考试难度包含高中数列（等差数列、等比数列）以及提高组难度的数学
  • 实际录取中，语文数学成绩差异不大，关键区分在编程能力与现学现考表现
  • 编程特别强 + 思维能力尚可 + 现学现考未崩的学生，录取率最高
  • 大量时间花在小奥刷题而编程练习不足的学生，反而落选

金刚校 C 的筛选特点

  • 重视学习习惯、习题表现、代码反馈等综合指标
  • 对编程当前水平要求不苛刻，更看重成长潜力与长期表现
  • 同一体系下的两个校区，筛选模式可能完全不同，需针对性准备

3.3 跟训 vs 培优班

跟训模式

  • 周期长，需要在体系内"熬资历、熬排名"
  • 适合需要逐步证明自己的学生
  • 部分学校跟训中不讲知识点，纯以考试筛选

培优班模式

  • 筛选更迅速，可跳过漫长跟训
  • 由学校教练轮转授课，双向选择
  • 培优班中被选中的学生直接进入洽谈签约流程

CHAPTER 04编程特长生的机遇与趋势

4.1 两大扩招驱动因素

因素一：竞赛成绩突破

今年有学生进入信息学国家队，这一成绩对相关学校产生了显著冲击，推动更多学校加大对信奥方向的投入与招生力度。

因素二：产业需求拉动

重庆引入了大量高新技术企业（如宇树科技落地大渡口），城市发展定位向高新技术产业倾斜，对相关人才的需求持续增长。

趋势判断 新信息学奥赛的发展路径将持续拓宽。各学校今年普遍加开了编程招生名额，编程特长生正处于近五年最好的窗口期。

4.2 生源结构特征

学编程的学生总量较少，加之各校的筛选机制，导致能留存下来的学生高度重合——到每所学校考试、上岸的往往是同一批人。这一特征意味着：

  • 编程特长生群体虽小，但优质学生在各校间具有高度可识别性
  • 对于尚未入场的学生家庭而言，现在进入仍有机会，但需要系统化的训练与规划

4.3 备考实例：原题命中率

数据参考 某机构寒假期间针对金刚校 B 考试进行了为期约一周的集训。该校考试共六道题，其中最难的第五、第六题（即集训作业原题）被成功命中两道。此外，多个金刚校的每周跟学中，该机构的思维月赛每周至少命中一道原题。但部分学生因未完成作业或未参加思维月赛，导致面对原题时无法作答。

CHAPTER 05信奥数学的核心地位

核心结论 编程学习中的大量卡点，根本原因并非学生智力不足，而是数学知识储备不够。编程涉及的数学内容大量超纲，学校课程无法覆盖，导致解题时频繁卡壳。

5.1 编程与数学的层级对应

编程等级 所需数学内容 难度层级
1—4级（普及组基础阶段） 方程、函数等初中数学内容 普及组数学
4级+（向提高组过渡） 普及组高段数学 + 部分高中内容 普及组→提高组衔接
7级+（提高组阶段） 高中数列（等差/等比）、大学线性代数 提高组数学

竞赛数学

5.2 推荐教材

《信息学奥赛之数学一本通》

  • 主编：林厚崇
  • 出版社：东南大学出版社
  • 版本：2016年出版（最新版）
  • 定位：金刚校教练推荐的必啃教材，冲省赛一等奖、提高组一等奖的必备数学基础
  • 注意：该书难度较高，小学高年级学生自学困难较大，建议由专业数学教师系统授课
警示 该教材是冲提高组一等奖和省赛一等奖的必啃内容。目前仅有一所金刚校配备了专职数学老师进行系统理论教学，其余学校均未完整覆盖——要么由竞赛教练"带到讲"，要么依赖学生自学。

5.3 某校考试验证

金刚校 A 寒假复试的"现学现考"环节中，直接考察了高中等差数列、等比数列以及提高组难度数学内容。这进一步印证了：在编程竞赛中，数学是不可回避的底层能力。

CHAPTER 06分阶段课程体系与培养路径

6.1 阶段划分与课程匹配

第一阶段：1—4级（基础打底层）

核心任务：编程基础 + 普及组数学补齐。加入约24节信奥数学课程，覆盖方程、函数等初中数学内容，为平稳过渡至四级以上做准备。此阶段不验证竞赛适配性，以基础建设为主。

第二阶段：4级+（匹配跟训/升学）

开始匹配学校的跟训活动，陆续进入升学推荐流程。同步增加针对性刷题课程（编程思维刷题班），针对目标学校考试模式进行专项训练。四级以下暂不开放升学规划。

第三阶段：6级+（培优冲刺层）

匹配信奥培优班，由学校教练轮转授课。走综合训练路线，通过高难度综合性竞赛题目快速提升能力，缩短签约周期。培优班中被选中者直接进入签约洽谈。

第四阶段：7级+（提高组深底层）

匹配提高组数学课程（高中数学+大学线性代数），使用林厚崇《信息学奥赛之数学一本通》教材，由专职数学教师系统授课。此阶段数学能力直接决定省赛成绩。

6.2 课程配套说明

编程等级 周课安排 数学内容 数学授课方
4级以前 1次编程 + 1次数学 普及组数学（函数、方程等） 金刚校竞赛教练
4级—6级 1—2次编程 + 针对性刷题 普及组高段 → 提高组衔接 金刚校竞赛教练
7级+ 培优班综合训练 提高组数学（高中+大学内容） 专职素锦数学老师
说明 所有市级以上学生均进入统一的推优群，推优信息统一公开发布。每月或每个周期公布各校计划，学生在班主任处报名后，由机构根据学生情况匹配学校，不匹配者沟通调整计划。

CHAPTER 07升学实战策略与建议

7.1 通过筛选的核心路径

朴实且有效的策略 过四级之后，更快签约的核心方法：多刷题、多练题、多针对性地练题。这是被多位金刚校总教练验证的训练逻辑——"如果刷了题还不行，只能说明刷的题还不够。"

7.2 推荐备考时间线

四年级及以前

打好编程基础 + 补齐普及组数学，不急于进入升学通道。目标：稳定通过四级。

五年级

进入各校跟训/内测/培优班，积累实战经验。五年级是关键窗口——部分学校五年级即有脱产机会。

六年级

最后机会。六年级学生必须全力以赴，所有选拔活动均为终局。需特别注意避免因疏忽（如误挂电话、错过通知）而丧失机会。

7.3 常见失误警示

信息层面

  • 误将学校复试通知当作骚扰电话挂断
  • 不了解不同学校考试时间冲突，未提前规划
  • 对目标学校的考试风格和内容缺乏调研

执行层面

  • 不按要求完成布置的寒假作业和思维月赛
  • 重心偏移：编程已进入关键阶段仍大量时间投入文化课
  • 不遵循规划建议，自行调整训练重心

7.4 给家长的核心建议

  1. 尊重专业规划：凡按规划执行的家庭，上岸概率极高；未上岸者多因未遵循安排
  2. 距离不是障碍：已有四川宜宾、达州等地学生每周乘高铁赴渝参加线下课程，线下学习对竞赛升学具有不可替代性
  3. 重视数学储备：编程卡壳的本质是数学知识不足，需在对应阶段系统补齐
  4. 重视刷题与作业：每周考试中原题命中率很高，未完成练习直接导致考场上无法作答
  5. 抓住时间窗口：当前是编程特长生近五年最好的升学窗口，各校扩招意愿强烈
总结 在学生具备基本资质的前提下，机构提供的课程体系、升学规划与信息渠道构成完整的支撑闭环。信息差是多数家庭面临的核心障碍——不了解学校怎么考、不知道怎么准备，往往输在起点。
