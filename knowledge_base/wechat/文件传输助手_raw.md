- [2026-02-22 17:33] me: 相对于 Splay 树，**无旋 Treap (FHQ Treap)** 的逻辑更加直观。它不依赖复杂的旋转操作，而是通过 **分裂 (Split)** 和 **合并 (Merge)** 两个核心操作来维护平衡。

在处理区间问题时，FHQ Treap 极其强大：我们可以直接把需要的区间“切”出来，处理完后再“粘”回去。

---

## FHQ Treap 核心逻辑

1. **优先级 (Priority)**：每个节点分配一个随机权值 `key`。通过维护这个随机权值的“堆”性质，确保树在期望意义下是平衡的（高度为 ）。
2. **按大小分裂 (Split by Size)**：这是处理序列问题的关键。我们将树分为两部分：前  个节点组成的树，以及剩下的节点组成的树。
3. **懒标记 (Lazy Tag)**：和 Splay 一样，区间翻转通过交换左右儿子实现，并配合 `pushdown` 延迟更新。

---

## C++ 代码实现

```cpp
#include <iostream>
#include <cstdio>
#include <ctime>
#include <cstdlib>

using namespace std;

const int MAXN = 100005;

struct Node {
    int l, r;       // 左右儿子下标
    int val;        // 节点存储的数值
    int key;        // 随机优先级（维护平衡的关键）
    int size;       // 子树大小
    int reverse;    // 翻转懒标记

    void init(int v) {
        val = v;
        key = rand(); // 分配随机权值
        size = 1;
        l = r = reverse = 0;
    }
} tr[MAXN];

int root, idx;

// 更新子树大小
void pushup(int u) {
    tr[u].size = tr[tr[u].l].size + tr[tr[u].r].size + 1;
}

// 下传翻转标记
void pushdown(int u) {
    if (tr[u].reverse) {
        swap(tr[u].l, tr[u].r);
        if (tr[u].l) tr[tr[u].l].reverse ^= 1;
        if (tr[u].r) tr[tr[u].r].reverse ^= 1;
        tr[u].reverse = 0;
    }
}

// 分裂操作：将以 u 为根的树，按前 k 个节点分裂成 x 和 y 两棵树
void split(int u, int k, int &x, int &y) {
    if (!u) {
        x = y = 0;
        return;
    }
    pushdown(u); // 分裂前必须下传标记，否则分裂出的子树顺序是错的
    
    // 如果左子树大小已经够了（或超过了）k，说明第 k 个节点在左边
    if (tr[tr[u].l].size < k) {
        x = u;
        // 递归分裂右子树，更新当前节点的右儿子
        split(tr[u].r, k - tr[tr[u].l].size - 1, tr[u].r, y);
    } else {
        y = u;
        // 递归分裂左子树，更新当前节点的左儿子
        split(tr[u].l, k, x, tr[u].l);
    }
    pushup(u);
}

// 合并操作：将 x 和 y 两棵树合并，返回新根（要求 x 中所有节点在 y 之前）
int merge(int x, int y) {
    if (!x || !y) return x + y;
    
    // 根据随机优先级 key 来决定谁做父节点（维护堆性质）
    if (tr[x].key < tr[y].key) {
        pushdown(x);
        tr[x].r = merge(tr[x].r, y); // x 在左边，递归合并它的右子树
        pushup(x);
        return x;
    } else {
        pushdown(y);
        tr[y].l = merge(x, tr[y].l); // y 在右边，递归合并它的左子树
        pushup(y);
        return y;
    }
}

// 插入新节点（文艺平衡树通常按 1~n 顺序插入，也可以直接 build）
void insert(int v) {
    tr[++idx].init(v);
    root = merge(root, idx);
}

// 中序遍历输出结果
void output(int u) {
    if (!u) return;
    pushdown(u);
    output(tr[u].l);
    printf("%d ", tr[u].val);
    output(tr[u].r);
}

int main() {
    srand(time(0)); // 初始化随机种子
    int n, m;
    scanf("%d%d", &n, &m);
    for (int i = 1; i <= n; i++) insert(i);

    while (m--) {
        int l, r;
        scanf("%d%d", &l, &r);
        int x, y, z;
        // 1. 剥离出区间 [l, r]
        // 先把前 r 个分出来到 x，剩下的到 z
        split(root, r, x, z);
        // 再从 x 中把前 l-1 个分出来到 x，剩下的到 y
        split(x, l - 1, x, y);
        
        // 2. 此时 y 树正好就是 [l, r] 区间，打上标记
        tr[y].reverse ^= 1;
        
        // 3. 按顺序粘回去：x 和 y 合并，再和 z 合并
        root = merge(merge(x, y), z);
    }

    output(root);
    return 0;
}

```

## 深度解析

### 1. 为什么 FHQ Treap 不需要旋转？

传统的 Treap 在插入时通过旋转维护堆性质。而 FHQ Treap 发现：只要我有 `split` 和 `merge`，我可以随意拆解树。在 `merge` 时，我们通过比较 `tr[x].key < tr[y].key` 来随机决定谁是谁的父亲，概率论保证了这样生成的树高度约为 。

### 2. 区间操作的精髓：三段式拆解

对于区间 ，我们的操作逻辑是：

* `split(root, r, x, z)`：把树切成“前  个”和“后面的”。
* `split(x, l-1, x, y)`：把“前  个”再切成“前  个”和“中间的”。
* **此时  树包含了原序列第  到第  个节点。**

这种方式比 Splay 的旋转提取更符合人类直觉。

### 3. `pushdown` 的位置

在无旋 Treap 中，`pushdown` 必须放在 `split` 和 `merge` 的递归入口处。

* 因为 `split` 和 `merge` 会改变子树结构，如果不及时交换左右儿子，你切出来的子树可能根本不是你想要的那个位置。
