- [2026-06-13 11:42] 张仁田: 在你的 macOS 上， mimo  默认安装路径一般在这几个地方，我帮你定位一下：
 
 
 
🔍 先找一下它在哪
 
在终端里执行下面这几条命令，就能直接找到安装位置：
 
bash
  
# 1. 查找系统里所有叫 mimo 的文件
find ~ -name "mimo" -type f 2>/dev/null

# 2. 查找系统里所有叫 mimo 的可执行文件
which -a mimo || echo "没在 PATH 里找到"

# 3. 看一下 PATH 里有没有它可能在的目录
echo $PATH
 
 
 
 
📂 常见的安装路径（按概率从高到低）
 
1. 用户目录下的 bin 目录（一键脚本默认）
plaintext
  
~/.local/bin/mimo
~/.mimo/bin/mimo
~/.cache/mimo/
 
 
一键安装脚本  curl -fsSL https://mimo.xiaomi.com/install | bash  通常会把二进制文件放在这里，并尝试把它加入  PATH 。
2. npm 全局安装路径（如果你用了 npm）
如果你之前尝试过  npm install -g @mimo-ai/cli ，它会在这里：
bash
  
# 查看 npm 全局路径
npm config get prefix
# 通常是 /usr/local 或 ~/.npm-global
# 对应的二进制文件在：
/usr/local/bin/mimo
~/.npm-global/bin/mimo
 
3. 配置和缓存目录
MiMo 的配置文件、会话数据会存在这里：
plaintext
  
~/.mimo/
~/.cache/mimo/
 
 
 
 
❓ 为什么你提示  command not found ？
 
大概率是安装脚本把  mimo  放到了  ~/.local/bin  这类目录，但这个目录没有被加入到你的  PATH  环境变量里，或者你的 shell（zsh/bash）没读到配置文件。
 
你可以这样解决：
 
bash
  
# 1. 先找到 mimo 的路径，比如是 ~/.local/bin/mimo
# 2. 临时测试运行
~/.local/bin/mimo

# 3. 永久加入 PATH（以 zsh 为例）
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

# 4. 再试试
mimo
