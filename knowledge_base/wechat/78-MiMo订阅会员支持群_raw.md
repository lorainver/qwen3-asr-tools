- [2026-05-29 19:54] 董得猛: The user is asking why MiMo V2.5 Pro requires `supportsImages: false` to work on CodeBuddy, otherwise it keeps getting the "No endpoints found that support image input" error.

Let me explain the technical reason clearly.
因为 CodeBuddy 的工作机制是这样的：

### 原理

当你开启 `supportsImages: true` 时，CodeBuddy 内部会这样处理：

1. **每次发送请求前**，它会先检查当前对话中**有没有附带图片**
2. 如果你**任何一次对话**（包括当前或历史消息）中有图片，它就会去找一个**支持图片输入的 API 端点**
3. 但如果 MiMo Token Plan 的 API **实际上不支持图片输入**，CodeBuddy 就找不到可用端点 → 直接报错

**哪怕你现在没有发图片，CodeBuddy 也会因为模型声称支持图片但找不到对应端点而报错。**

---

### 简单类比

就像你告诉 CodeBuddy："我的车能飞"，但实际去飞的时候，飞机场（API 端点）根本不存在，于是报错。

`supportsImages: false` = 如实告诉 CodeBuddy："我的车只能跑，不能飞"，这样 CodeBuddy 就不会尝试找飞机场了。

---

### 本质原因

**MiMo Token Plan 的 OpenAI 兼容端点本身就不支持图片格式**，这是 API 端的限制，不是配置问题。MiMo 模型本身可能有多模态能力，但 Token Plan 的 API 接口没有开放这个功能，所以 CodeBuddy 无法调用。

所以目前没有其他办法，必须保持 `supportsImages: false`。等 MiMo 后续更新 API 支持图片端点了，再改回来就行。
